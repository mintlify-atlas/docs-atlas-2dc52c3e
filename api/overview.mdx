The nvitop Python API lets you query live GPU state, enumerate processes, and collect time-series resource metrics directly from Python — no shell parsing, no subprocess calls. Everything is exposed through a small set of classes and functions that ship inside the `nvitop` package, so no extra installation step is required beyond `pip install nvitop`.

<Note>
The API requires the NVIDIA Management Library (NVML) to be present at runtime. NVML is bundled with every NVIDIA display driver. Import errors are raised as `NVMLError` subclasses, not plain `ImportError`.
</Note>

## Installation

```bash
pip install nvitop
```

All public symbols are re-exported from the top-level `nvitop` namespace:

```python
from nvitop import (
    Device, PhysicalDevice, MigDevice,
    CudaDevice, CudaMigDevice,
    parse_cuda_visible_devices, normalize_cuda_visible_devices,
    HostProcess, GpuProcess, command_join,
    ResourceMetricCollector, take_snapshots, collect_in_background,
    NA, NaType, Snapshot,
    KiB, MiB, GiB, TiB, PiB,
    bytes2human, human2bytes, timedelta2human,
    utilization2string, colored, set_color, boolify,
    ttl_cache,
    NVMLError, nvmlCheckReturn,
)
```

## Quick example

The following ten-line script prints live GPU stats and the processes running on each device:

```python
from nvitop import Device

devices = Device.all()          # list[PhysicalDevice]
for device in devices:
    processes = device.processes()   # dict[int, GpuProcess]
    print(device)
    print(f'  Used memory : {device.memory_used_human()}')
    print(f'  GPU util    : {device.gpu_utilization()}%')
    print(f'  Processes   : {len(processes)}')
    for pid, proc in sorted(processes.items()):
        print(f'    [{pid}] {proc.command()} — {proc.gpu_memory_human()}')
```
