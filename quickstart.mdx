---
title: "Get up and running with nvitop in minutes"
description: "Install nvitop, launch the interactive monitor, run a one-shot status query, write your first Python API script, and select GPUs automatically with nvisel."
---

This page walks you through the most common nvitop workflows from a fresh install to your first Python API script. Each step builds on the previous one, but you can jump to any section if you already have nvitop installed.

<Steps>
  <Step title="Install nvitop">
    The fastest way to run nvitop without touching your global Python environment is with `uvx`:

    ```bash
    uvx nvitop
    ```

    For a permanent install available in every shell session:

    ```bash
    pip3 install --upgrade nvitop
    ```

    See [Installation](/installation) for conda, pipx, and source install options.
  </Step>

  <Step title="Run a one-shot status query">
    Use the `-1` flag to print GPU and process status once and exit — the same idea as `nvidia-smi`, but with more detail and color:

    ```bash
    nvitop -1
    ```

    Useful flags for the one-shot mode:

    ```bash
    # Show only GPU 0 and GPU 1
    nvitop -1 -o 0 1

    # Show only devices listed in CUDA_VISIBLE_DEVICES
    nvitop -1 -ov

    # Show only processes with a compute context (type C or C+G)
    nvitop -1 -c
    ```

    <Tip>
      `-1` is significantly faster than monitor mode and uses minimal resources. Use it in scripts or cron jobs where you need a quick status snapshot.
    </Tip>
  </Step>

  <Step title="Launch the interactive monitor">
    Run `nvitop` without `-1` to enter monitor mode — a live, continuously-updating TUI:

    ```bash
    nvitop
    ```

    The display mode defaults to `auto`, which adjusts the layout based on terminal size. You can choose a mode explicitly:

    ```bash
    nvitop -m full     # detailed full-width layout
    nvitop -m compact  # condensed layout for narrow terminals
    nvitop -m auto     # automatic (default)
    ```

    **Essential keybindings:**

    | Key | Action |
    |---|---|
    | `h` or `?` | Open the help screen |
    | `q` | Quit and return to the terminal |
    | `a` / `f` / `c` | Switch to auto / full / compact mode |
    | `t` | Toggle tree-view of GPU processes |
    | `e` | Show environment variables for the selected process |
    | `Enter` | Open per-process metrics graphs |
    | `T` / `K` | Send SIGTERM / SIGKILL to the selected process |
    | `,` / `.` | Move the sort column left / right |

    <Tip>
      To kill or terminate a misbehaving process, navigate to it with the arrow keys and press `T` (terminate) or `K` (kill). Using the tree-view screen (`t`) first helps you identify the correct process before sending a signal.
    </Tip>
  </Step>

  <Step title="Check all available options">
    Run `nvitop --help` to see every command-line option:

    ```text
    usage: nvitop [--help] [--version] [--once | --monitor [{auto,full,compact}]]
                  [--interval SEC] [--ascii] [--colorful] [--force-color] [--light]
                  [--gpu-util-thresh th1 th2] [--mem-util-thresh th1 th2]
                  [--only INDEX [INDEX ...]] [--only-visible]
                  [--compute] [--only-compute] [--graphics] [--only-graphics]
                  [--user [USERNAME ...]] [--pid PID [PID ...]]

    An interactive NVIDIA-GPU process viewer.

    options:
      --help, -h            Show this help message and exit.
      --version, -V         Show nvitop's version number and exit.
      --once, -1            Report query data only once.
      --monitor [{auto,full,compact}], -m [{auto,full,compact}]
                            Run as a resource monitor.
      --interval SEC        Process status update interval in seconds. (default: 2)
      --ascii, --no-unicode, -U
                            Use ASCII characters only.
    ```

    See the [CLI reference](/cli/nvitop) for the full option table.
  </Step>

  <Step title="Write your first Python API script">
    nvitop exposes a Python API you can import directly. The `Device` class is the entry point for querying GPU status:

    ```python
    from nvitop import Device

    devices = Device.all()  # returns a list of all NVML-enumerated devices

    for device in devices:
        processes = device.processes()  # dict mapping PID -> GpuProcess
        sorted_pids = sorted(processes.keys())

        print(device)
        print(f'  - Fan speed:       {device.fan_speed()}%')
        print(f'  - Temperature:     {device.temperature()}C')
        print(f'  - GPU utilization: {device.gpu_utilization()}%')
        print(f'  - Total memory:    {device.memory_total_human()}')
        print(f'  - Used memory:     {device.memory_used_human()}')
        print(f'  - Free memory:     {device.memory_free_human()}')
        print(f'  - Processes ({len(processes)}): {sorted_pids}')
        for pid in sorted_pids:
            print(f'    - {processes[pid]}')
        print('-' * 80)
    ```

    To use CUDA device ordering (respecting `CUDA_VISIBLE_DEVICES`) instead of NVML ordering, replace `Device.all()` with `Device.cuda.all()`.

    <Note>
      `Device.all()` uses the NVML device index. `Device.cuda.all()` uses the CUDA device index, which respects the `CUDA_VISIBLE_DEVICES` and `CUDA_DEVICE_ORDER` environment variables. The two orderings may differ.
    </Note>
  </Step>

  <Step title="Select GPUs automatically with nvisel">
    `nvisel` is a companion CLI tool that selects the best available GPUs based on memory and utilization constraints and prints the result as a comma-separated list suitable for `CUDA_VISIBLE_DEVICES`.

    ```bash
    # Select 2 GPUs with at least 8 GiB of free memory each
    export CUDA_VISIBLE_DEVICES="$(nvisel -c 2 -f 8GiB)"

    # Select up to 4 GPUs with GPU utilization under 60%
    export CUDA_VISIBLE_DEVICES="$(nvisel -C 4 --max-gpu-utilization 60)"

    # Output UUIDs instead of integer indices
    export CUDA_VISIBLE_DEVICES="$(nvisel -O uuid -c 2 -f 5GiB)"
    ```

    You can also call `nvisel` from Python:

    ```python
    import os
    from nvitop import select_devices

    os.environ['CUDA_VISIBLE_DEVICES'] = ','.join(
        select_devices(format='uuid', min_count=4, min_free_memory='8GiB')
    )
    ```

    See the [nvisel reference](/cli/nvisel) for all available constraints and formatting options.
  </Step>
</Steps>
