nvitop is an interactive NVIDIA-GPU process viewer and resource monitor that goes far beyond `nvidia-smi`. It provides a colorful, continuously-updating terminal interface for monitoring GPU devices and processes, a CLI tool for automatically selecting CUDA devices, and a rich Python API for building your own monitoring integrations.

## What makes nvitop different

nvitop is built around three principles: **richness**, **efficiency**, and **integrability**. Unlike `nvidia-smi`, which prints a static snapshot, nvitop runs as a live monitor with bar charts, history graphs, process trees, and keyboard-driven interactions. Unlike other GPU monitors written in C, nvitop is pure Python — easy to install and easy to embed in your own tools.

## Get started

<Steps>
  <Step title="Install nvitop">
    The fastest way is via `uvx` or `pipx` — no virtual environment needed:
    ```bash
    uvx nvitop
    ```
    Or install permanently with pip:
    ```bash
    pip install nvitop
    ```
  </Step>
  <Step title="Launch the monitor">
    Run `nvitop` to enter interactive monitor mode and see all your GPUs in real time:
    ```bash
    nvitop
    ```
    Press `h` for the help screen, `q` to quit.
  </Step>
  <Step title="Use the Python API">
    Query device status in your own scripts with just a few lines:
    ```python
    from nvitop import Device

    for device in Device.all():
        print(device, device.gpu_utilization(), device.memory_used_human())
    ```
  </Step>
  <Step title="Select GPUs automatically">
    Use `nvisel` to pick the best available GPUs before starting a training job:
    ```bash
    export CUDA_VISIBLE_DEVICES="$(nvisel -c 2 -f 8GiB)"
    ```
  </Step>
</Steps>

<Note>
  nvitop requires an NVIDIA GPU with the NVML library (included with the NVIDIA display driver). It runs on both Linux and Windows with Python 3.8+.
</Note>
