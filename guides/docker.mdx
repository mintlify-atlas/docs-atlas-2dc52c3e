---
title: "Running nvitop inside Docker containers"
description: "Use nvitop with nvidia-container-toolkit to monitor GPU processes from inside a Docker container, including the Grafana dashboard quickstart via Docker Compose."
---

Docker containers run in isolated namespaces by default, which means a containerized `nvitop` can only see processes that belong to the container itself. Two flags are required to give `nvitop` full visibility: `--runtime=nvidia --gpus=all` to expose the NVIDIA devices, and `--pid=host` to share the host PID namespace so that all GPU processes on the machine are visible.

## Prerequisites

- [nvidia-container-toolkit](https://github.com/NVIDIA/nvidia-container-toolkit) installed and configured on the host
- Docker Engine with the NVIDIA runtime enabled

## Running the prebuilt image

The fastest way to run `nvitop` in a container is to pull the prebuilt image from the GitHub Container Registry:

```bash
docker run -it --rm --runtime=nvidia --gpus=all --pid=host ghcr.io/xuehaipan/nvitop:latest
```

<Warning>
  The `--pid=host` flag is required. Without it, `nvitop` can only see processes
  running inside the container and will not show the full picture of GPU usage on
  the host.
</Warning>

The flags explained:

| Flag | Purpose |
|------|---------|
| `--runtime=nvidia` | Activates the NVIDIA container runtime so GPU devices are accessible |
| `--gpus=all` | Passes all host GPUs into the container |
| `--pid=host` | Shares the host PID namespace so all GPU processes are visible |
| `-it` | Allocates a pseudo-terminal, required for `nvitop` monitor mode |
| `--rm` | Removes the container automatically when you exit |

## Building the image yourself

If you want to build from source, clone the repository and use the provided `Dockerfile`:

```bash
git clone --depth=1 https://github.com/XuehaiPan/nvitop.git
cd nvitop
docker build --tag nvitop:latest .
docker run -it --rm --runtime=nvidia --gpus=all --pid=host nvitop:latest
```

The `Dockerfile` sets up a Python virtual environment inside `ubuntu:latest`, installs `nvitop`, and uses `python3 -m nvitop` as the container entrypoint.

## Grafana dashboard via Docker Compose

If you only need to set up the full Grafana monitoring stack (Prometheus + Grafana + nvitop-exporter), use the Docker Compose configuration bundled in the `nvitop-exporter/grafana` directory:

```bash
docker compose --project-directory=nvitop-exporter/grafana up --build --detach
```

This starts three services and makes the Grafana dashboard available at [http://localhost:3000](http://localhost:3000).
